# V0 Technical Stack

## Summary

The V0 uses a TypeScript monorepo with a Vite/React PWA frontend, a NestJS REST API backend, Neon Postgres, Drizzle, Better Auth, Cloudflare R2, and OpenAPI-generated frontend API clients.

## Runtime and hosting

| Area | Decision |
| --- | --- |
| Frontend | Vite + React + TypeScript |
| Frontend hosting | Cloudflare Pages |
| Backend | NestJS + TypeScript |
| Backend hosting | Fly.io |
| Database | Neon Postgres |
| File storage | Cloudflare R2 |
| Package manager | pnpm workspaces |
| Monorepo orchestration | Turborepo |

## Monorepo layout

```txt
apps/
  web/              # Vite + React PWA
  api/              # NestJS REST API
packages/
  database/         # Drizzle schema, migrations, seed scripts
  contracts/        # Zod schemas shared by API and web
  config/           # shared tsconfig/biome/build config if needed
  types/            # shared TypeScript-only types if needed
docs/
```

## Backend

- NestJS exposes a REST API.
- API documentation and contract are exposed through OpenAPI/Swagger.
- Input validation uses Zod, not class-validator.
- Zod schemas are the source for DTO validation and OpenAPI generation where possible.
- Backend logs use Pino as structured JSON logs to stdout on Fly.io.
- Background jobs run inside the NestJS backend scheduler for V0.
- Critical jobs use Postgres locks to avoid duplicate execution if the backend scales to multiple instances.
- No Redis, BullMQ, or external cache in V0.

## Frontend

- Vite + React + TypeScript.
- TanStack Router for routing.
- TanStack Query for server state and cache.
- React Hook Form + Zod resolver for forms.
- OpenAPI-generated TypeScript client using `openapi-typescript` + `openapi-fetch`.
- No Ky in V0.
- ReUI is the primary component library.
- shadcn/ui is used as fallback when ReUI does not provide a needed component.
- Tailwind is the styling foundation.
- Huge Icons is the icon set.
- No charting library in V0.

## Auth

- Better Auth.
- Email/password login.
- Session cookie is httpOnly.
- CORS uses `credentials: true`.
- Local development uses localhost origins and cookies without `Secure` or explicit `Domain`.
- Production uses app/API subdomains and secure cookies.

Expected production domains:

- frontend: `https://app.tatamiq.com`
- API: `https://api.tatamiq.com`

Important environment variables:

- `APP_URL`
- `API_URL`
- `COOKIE_DOMAIN`
- `COOKIE_SECURE`
- `CORS_ORIGIN`

## Database

- Neon Postgres.
- Drizzle ORM/query builder.
- Drizzle schema and migrations live in `packages/database`.
- Neon branches separate staging and production in V0.
- Tenant isolation is by academy from the start.

## File storage

- Cloudflare R2.
- Separate buckets for staging and production.
- Used for academy logos, student photos, and Pix receipts.
- Frontend uploads directly to R2 using signed URLs generated by the backend.
- Backend owns authorization, metadata creation, and persistence of file references.

## QR attendance tokens

- QR Code uses signed stateless tokens generated by the backend.
- Tokens include the class session, validity window, and token version/nonce data as needed.
- Tokens rotate every 30 seconds in the UI.
- The previous token is accepted for a short tolerance window.
- Tokens are not persisted every 30 seconds.
- Attendance records are still persisted when the student confirms presence.

## Date and time

- Day.js with `utc` and `timezone` plugins.
- Date/time helpers should be centralized in one module.
- Default V0 timezone: `America/Sao_Paulo`.

## Testing

- Vitest for unit tests and lightweight integration tests.
- Testing Library for React component tests.
- Playwright for E2E flows.

Critical E2E flows:

- instructor login;
- student invite acceptance;
- class start and QR attendance;
- manual attendance and invalidation;
- monthly fee Pix receipt submission and verification;
- graduation promotion.

## Quality and CI/CD

- Biome for lint and formatting.
- Husky + lint-staged to run Biome on changed files.
- GitHub Actions for CI.
- CI should run install, typecheck, lint, tests, and builds for apps/packages.
- Deploy automation should exist from early V0.
- Environments: staging and production.

## Product analytics and logs

- PostHog for basic product analytics in frontend and backend.
- No PostHog session replay in V0.
- Avoid sending sensitive data to PostHog.
- Pino logs structured JSON to stdout on Fly.io.

Minimum PostHog events:

- instructor created student;
- instructor created class group;
- instructor started class;
- student confirmed attendance;
- instructor invalidated attendance;
- monthly fee generated;
- student submitted Pix receipt;
- instructor approved Pix receipt;
- instructor rejected Pix receipt;
- graduation promotion registered;
- student invite accepted.
