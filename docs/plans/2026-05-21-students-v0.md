**Status:** Completed

**Completed on:** 2026-05-21

**Implementation commits:**

## Goal

Implement the complete V0 student registration slice for the instructor area. This makes `/students` a real operational screen backed by tenant-scoped API endpoints and database tables.

This slice covers student records, guardian data for minors, billing defaults, and initial graduation state. CSV import/export and class-group enrollment are intentionally left for later slices.

## Scope

### Included

- Tenant-scoped `students` table using Better Auth `organization.id` as `organization_id`.
- Tenant-scoped `student_guardians` table.
- Protected API endpoints for owner/instructor users.
- OpenAPI contracts for student operations.
- Web `/students` page replacing the placeholder.
- Create, edit, list, inactivate, and reactivate students.
- Validation for minor students requiring guardian data.
- Initial billing defaults per student.
- Initial graduation state per student.

### Deferred

- CSV import/export.
- Class group enrollment.
- Student app access and `StudentAccess`.
- Photo upload.
- Full graduation history and promotion workflow.
- Monthly fee generation.

## Domain decisions

- **Aluno** belongs to exactly one **Academia**.
- Backend derives the academy from `session.session.activeOrganizationId`; clients never send `organizationId`/`academyId`.
- **Aluno Ativo** appears in future attendance and billing workflows.
- **Aluno Inativo** is preserved for history and excluded from future attendance/billing workflows.
- Inactivation sets `inactive_at`.
- Reactivation clears `inactive_at`.
- A minor student is any student under 18 on the current date.
- Minor students require one guardian with name and phone.
- Guardian email is optional.
- Billing fields are optional in this slice, but validated when present.
- Graduation state is required at creation with simple enums, not a full history model yet.

## Data model

### `students`

- `id` text primary key.
- `organization_id` text not null references `organization.id`.
- `name` text not null.
- `birth_date` date not null.
- `enrollment_date` date not null.
- `status` text not null: `active | inactive`.
- `inactive_at` timestamp nullable.
- `phone` text nullable.
- `email` text nullable.
- `monthly_amount_in_cents` integer nullable.
- `monthly_due_day` integer nullable, 1-31.
- `current_belt` text not null.
- `current_degree` integer not null, 0-4.
- `graduation_path` text not null: `adult | child`.
- `created_at` timestamp not null.
- `updated_at` timestamp not null.

### `student_guardians`

- `id` text primary key.
- `student_id` text not null references `students.id` on delete cascade.
- `name` text not null.
- `phone` text not null.
- `email` text nullable.
- `relationship` text nullable.
- `created_at` timestamp not null.
- `updated_at` timestamp not null.

## API

All endpoints require an authenticated session with active organization and owner role.

- `GET /students`
  - Returns students for the active academy.
  - Supports optional `status=active|inactive|all`.
- `POST /students`
  - Creates a student in the active academy.
  - Requires guardian when the birth date indicates the student is a minor.
- `GET /students/:id`
  - Returns one student from the active academy.
- `PATCH /students/:id`
  - Updates student and guardian data.
  - Keeps tenant isolation.
- `POST /students/:id/inactivate`
  - Sets status to inactive and `inactive_at`.
- `POST /students/:id/reactivate`
  - Sets status to active and clears `inactive_at`.

## Web UI

`/students` should provide:

- Header with page title and create button.
- Filter chips/counts for active/inactive students.
- Empty state for no students.
- Table or card list with name, status, age, enrollment date, belt/degree, and billing summary.
- Create/edit form in a modal or sheet.
- Guardian section shown for minors and available manually for all students.
- Clear validation messages in Portuguese.
- Inactivate/reactivate actions.
