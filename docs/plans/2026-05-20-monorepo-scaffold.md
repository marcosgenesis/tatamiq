## Goal

Create the first working code scaffold for Tatamiq as a TypeScript monorepo. The scaffold should validate the chosen stack end-to-end without implementing core product domain flows yet.

## Decisions

- Use real working apps, not empty placeholders.
- Use Node 22 LTS.
- Use pnpm 10.x with `packageManager` pinned in the root `package.json`.
- Use pnpm workspaces and Turborepo.
- Use official generators for the initial Vite and NestJS apps, then adjust the generated output.
- Use Conventional Commits and enforce them with commitlint.

## Package layout

```txt
apps/
  web/              # @tatamiq/web — Vite + React + TypeScript
  api/              # @tatamiq/api — NestJS + TypeScript
packages/
  database/         # @tatamiq/database — Drizzle schema, migrations, seed scripts
  contracts/        # @tatamiq/contracts — Zod schemas, OpenAPI artifact, generated client
  config/           # @tatamiq/config — shared TypeScript configs
docs/
```

Do not create `@tatamiq/types` yet. Shared types should come from contracts or database until a concrete need appears.

## Root tooling

Add root configuration for:

- Turborepo;
- pnpm workspaces;
- TypeScript base scripts;
- Biome;
- Husky;
- lint-staged;
- commitlint;
- GitHub Actions CI;
- Docker Compose with local Postgres;
- root `.env.example`.

Standard scripts:

- `dev`
- `build`
- `typecheck`
- `lint`
- `format`
- `test`
- `openapi:generate`

## Shared config package

`@tatamiq/config` should provide TypeScript configs:

- `tsconfig.base.json`
- `tsconfig.web.json`
- `tsconfig.node.json`

Biome stays configured at the repo root, not exported from the config package.

## Web app

Create `apps/web` as `@tatamiq/web` using Vite + React + TypeScript.

Configure in the scaffold:

- Tailwind;
- TanStack Router;
- TanStack Query;
- OpenAPI generated client consumption from `@tatamiq/contracts`;
- environment file example at `apps/web/.env.example`;
- initial page that calls the API and displays:
  - API health status;
  - demo academy from the database.

Do not configure ReUI or shadcn/ui in this scaffold step. They come next.

## API app

Create `apps/api` as `@tatamiq/api` using NestJS + TypeScript.

Configure in the scaffold:

- Pino logging;
- CORS ready for credentialed requests;
- Swagger/OpenAPI;
- Zod validation plumbing;
- environment file example at `apps/api/.env.example`;
- Dockerfile for Fly.io-compatible deployment;
- public `GET /health` endpoint;
- public `GET /academies/demo` endpoint to validate API + database + Drizzle.

Do not implement Better Auth in this scaffold step. It is the next functional slice.

## Database package

Create `packages/database` as `@tatamiq/database`.

Configure:

- Drizzle;
- local Postgres connection via Docker Compose;
- schema with initial `academies` table only;
- first migration;
- seed script that creates an `Academia Demo` record;
- scripts:
  - `db:generate`
  - `db:migrate`
  - `db:studio`
  - `db:seed`

Do not create `users` or auth tables yet. Better Auth may impose or influence that structure.

## Contracts package

Create `packages/contracts` as `@tatamiq/contracts`.

Configure:

- Zod schemas for the initial endpoints:
  - Health response;
  - Academy demo response;
- `packages/contracts/openapi.json` as the generated OpenAPI artifact;
- generated TypeScript client in this package;
- OpenAPI client generation using `openapi-typescript` + `openapi-fetch`.

Generation should use an OpenAPI file generated by the backend, not a running API URL. This avoids requiring the API to be running in CI.

## CI

GitHub Actions should run on pull requests and pushes to `main`:

- install dependencies;
- lint;
- typecheck;
- test;
- build.

## Acceptance criteria

The scaffold is complete when:

- [x] `pnpm install` works;
- [x] `pnpm dev` starts web and API through Turborepo;
- [x] local Postgres starts with Docker Compose;
- [x] Drizzle migration runs;
- [x] seed creates `Academia Demo`;
- [x] `GET /health` works;
- [x] `GET /academies/demo` reads from Postgres;
- [x] web app displays API health and demo academy;
- [x] OpenAPI file is generated into `packages/contracts/openapi.json`;
- [x] frontend client is generated from the OpenAPI file;
- [x] `pnpm lint`, `pnpm typecheck`, `pnpm test`, and `pnpm build` pass;
- [x] Husky + lint-staged are installed;
- [x] commitlint enforces Conventional Commits;
- [x] GitHub Actions CI is present;
- [x] Playwright smoke test validates API health, demo academy, and web status dashboard.
